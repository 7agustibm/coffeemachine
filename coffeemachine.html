Coffee Machine Project

Introduction

This mini project is used to learn TDD.

Of course, in your career, you have implemented more complicated stuff than a simple coffe machine that takes orders. But to make this mini project more interesting here are simple rules you must follow:

  1. All production code is written to make a failing test pass
  2. Do the simplest thing that could work for the current iteration

Are you ready? So read your assignement below and start the first iteration. Your product owner is waiting for you!

Project

The Coffee Machine Project is about the implementation of the engine that receives commands from the user and that dispatches commands to the drink maker according to the drink maker protocol.

This project starts simple and grow in added features through the iterations.

Drink maker protocol

The drink maker receives commands to make the drinks and can deliver info messages to the user. The instructions it receives follow this format:

T:1:0 ==> Drink maker makes 1 tea with 1 sugar and a stick
H:: ==> Drink maker makes 1 chocolate with no sugar
	(and therefore no stick)
C:2:0 ==> Drink maker makes 1 coffee with 2 sugars and a stick
M:message content ==> Drink maker will forward the message 
			to the coffee machine console

Requirements

All you need is:

  • A testing framework (junit, rspec, ...)
  • A mocking framework (mockito, ...)
  • A passion for tested code ;)
